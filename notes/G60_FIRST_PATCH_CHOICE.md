# G60 First Patch Choice

## Goal

Choose a single first local G60 patch to serve as the testbed for framed transport notions.

## Why this matters

The bridge work will sprawl unless one local carrier is chosen explicitly.

The toy layer succeeded because the local site was explicit.
The G60 lift needs the same discipline.

## Selection criterion

The first patch should be the smallest local G60 object that supports:

- an incoming oriented continuation
- at least two distinguishable local continuation classes
- a local chart choice
- a short return-capable neighborhood

## Chosen first target

For v0.1, the preferred first target is:

- an **oriented chamber-centered local adjacency patch**

That means:

- choose one chamber as the local carrier
- include its immediately adjacent oriented continuations
- retain enough local incidence structure to define a chart
- retain enough neighborhood to trace a short framed path

## Why this is the right first choice

This choice is small enough to stay concrete, but rich enough to support:

- local chart-relative exits
- parity-like continuation flips
- displacement-like framed accumulation
- short concatenations and returns

It is better than:
- trying to use the full G60 object at once
- using a completely abstract unnamed site
- jumping directly to global loops

## Working interpretation

The chamber-centered patch should answer:

- what counts as arrival?
- what are the locally available continuations?
- which continuations are chart-left-like and chart-right-like?
- what is the smallest local reversal?
- what is the smallest short framed return?

## Immediate next step

Describe the chamber-centered patch explicitly enough that a first framed readout can be defined on it.

## Strongest sentence

The first G60 framed testbed will be an oriented chamber-centered local adjacency patch.
