# G60 Minimal Framed Patch Candidate

## Goal

Identify the smallest nontrivial G60 local patch on which framed transport notions can be meaningfully tested.

## Requirements for the patch

A valid first G60 framed patch should support:

- a local chart choice
- at least two distinguishable local continuation directions
- a notion of local reversal or orientation change
- a short closed or near-closed return path
- explicit local incidence data

## Minimal questions the patch must answer

For a local G60 site:

- what counts as the local state carrier?
- what counts as the incoming incidence?
- what are the candidate local continuations?
- how is a local chart chosen?
- what is the local analogue of chart-left / chart-right?
- what is the smallest return path through adjacent sites?

## Candidate object types

Possible G60 local carriers to inspect:

- chamber
- oriented chamber
- flag neighborhood
- local incidence packet
- edge/face incidence site
- small oriented adjacency patch

## Preferred first target

The best first patch is not the whole G60 object.

The best first patch is:

- one local site with explicit oriented continuation classes, or
- two adjacent local sites with a short return-capable neighborhood

## Why this matters

The toy layer succeeded because the local objects were explicit.

The G60 lift will only work if the local patch is explicit enough to support:

- chart-relative exit
- framed transition sign
- local continuation class
- short return receipt

## Strongest sentence

The first G60 framed patch should be the smallest incidence neighborhood on which chart-relative continuations and a short framed return can both be defined.
