Harness README

This page is the Sprint 2 witness harness. Sprint 1 established a trustworthy console over the real witness machine. Sprint 2 adds a minimal structural map: the witness cycle is now exposed as a small graph without introducing camera, 3D, or speculative geometry.

The Python witness machine remains the source of truth. The browser does not invent machine semantics. It asks the API for a state payload and a structural payload, then renders either a console readout or a minimal graph view.

What changed in Sprint 2

  • A new structure endpoint maps a state to nodes, edges, and assembly metadata.
  • A new graph view renders the normalized witness cycle as a closed six-node ring.
  • Diads, couplers, and payload-carrying edges are distinguished visually.
  • The console remains the primary debugging instrument.

Structural interpretation

The normalized witness cycle is interpreted as the ordered register W, X, Y, Z, T, I. The first structural layer is the hexagonal witness with diads WX, YZ, TI and couplers XY, ZT, IW. The payload sits at T.

How to use it

  • Use τ to advance around the machine while phase remains fixed.
  • Use μ to flip between subjective and objective sheets.
  • Switch between Console and Graph views to compare readout and structure.
  • Use autoplay to observe local orbit behavior.

Why this is still minimal

Sprint 2 does not add perspective, camera, 3D, or animation beyond state stepping. The graph view is intentionally simple. Its job is only to make the witness cycle legible as structure while keeping the machine payload canonical.
