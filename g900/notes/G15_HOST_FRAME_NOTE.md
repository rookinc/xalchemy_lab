# G15 as Host Frame for G30, AT4val[60,6], and G900

## Status
Working note

## Core claim

We treat **G15** not as the final object of interest, but as the **rigid host frame**: the minimal lawful chambered scaffold that preserves orientation, supports admissible transport, and provides the positional grammar within which larger constructions become legible.

In this role, G15 houses:

- **G30** as an extended or doubled circulation
- **AT4val[60,6]** as a fuller structured inhabitant
- **G900** as a large admissible population visible through the lens

The point is architectural, not merely numerical.

G15 is not important because it is smaller.  
G15 is important because it is the **control frame by role**.

## 1. Informal picture

The intended hierarchy is:

\[
G15 \rightarrow G30 \rightarrow \mathrm{AT4val}[60,6] \rightarrow G900
\]

but this arrow should not be read as “bigger graph after smaller graph” in a naive size sense.

Instead:

- **G15** supplies the rigid chamber scaffold
- **G30** supplies richer transport or doubled routing
- **AT4val[60,6]** supplies a valence-complete finite inhabitant
- **G900** supplies a large lawful emission or population generated within the same admissibility regime

Thus G15 is the house, not the furniture.

## 2. Terminology

### 2.1 Host frame
A **host frame** is a finite rigid positional scaffold equipped with a lawful transition grammar.

It provides:

- a finite chamber structure
- a stable orientation system
- a distinguished notion of admissible local motion
- a closure grammar for extended traversals

### 2.2 Occupant
An **occupant** is a richer graph, transport population, or lens-visible structure whose organization is governed by the host frame.

An occupant need not be a literal subgraph in the naive graph-theoretic sense.  
It may instead be induced, transported, distributed, or rendered through the host grammar.

### 2.3 Transport grammar
A **transport grammar** is a finite operator system acting on oriented states.

In the present notes, the primitive operators are:

- `lift`
- `left`
- `right`
- `move`

These are not merely drawing commands.  
They are the native kinematics of the host.

### 2.4 Lens-visible population
A **lens-visible population** is the observable image of admissible action under a chosen compression or visibility rule.

For the current project language, admissible action belongs to the generator, while legibility belongs to the lens.

## 3. Working claim about G15

We propose:

> G15 is the rigid positional scaffold whose oriented transition grammar supports the embedding, circulation, transport, and lawful growth of G30, AT4val[60,6], and G900.

Equivalently:

> G15 is the admissibility frame.

Or in shorter project language:

> G15 is the rigid grammar of place.

This means that the larger constructions are not being treated as unrelated independent figures.  
They are treated as progressively richer occupants of one and the same lawful housing system.

---

## 4. Why G15 matters

The role of G15 is to preserve the following simultaneously:

1. **orientation**
   - left/right distinctions remain meaningful
   - lift is coherent
   - motion is not arbitrary

2. **chambered rigidity**
   - the scaffold is not merely a loose graph
   - it behaves like a finite system of rooms, registers, or sheets

3. **closure**
   - extended motion words can return lawfully
   - cyclic structure is supported by the frame

4. **scalability of inhabitance**
   - larger populations can be expressed without losing the local motion grammar

In this sense, G15 is the smallest structure in the stack that still behaves like a lawful house.

## 6. Structural distinction: host versus inhabitant

A key distinction is:

\[
\text{host role} \neq \text{occupant role}
\]

A host frame provides:
- admissibility
- orientation
- chamber logic
- positional discipline

An occupant provides:
- populated structure
- extended transport
- visible pattern
- fuller coherence manifestations

So even if some larger object contains or resembles pieces of G15, that is not the main point.

The main point is that **G15 governs the conditions under which the larger object is properly organized and read**.

## 7. Relation to the traversal grammar

The earlier walk notation already suggests this.

If a traversal state is written as:

\[
s = (v,\sigma,h)
\]

where:
- \(v\) is ambient location
- \(\sigma\) is sheet/layer/sign
- \(h\) is heading

then the operators

- \(U\) = lift
- \(L\) = left
- \(R\) = right
- \(M\) = move

act not on bare vertices, but on **oriented states**.

That is exactly what a host frame should support.

Hence the traversal grammar is evidence that G15 is more naturally interpreted as a rigid orientation-bearing housing system than as a mere finite graph drawn in the plane.

## 8. Circle squared reading

In this framework, the earlier “circle squared” intuition can be sharpened.

The host is chambered, orthogonal, and rigid.  
The transported walk is cyclic, returning, and rounded in behavior.

So the phenomenon is not literal Euclidean squaring of a circle.  
It is:

> cyclic closure realized through a rigid chambered scaffold

or more compactly:

> round return generated by lawful corners

This strengthens the case that the host frame language is more appropriate than a purely static graph language.

## 9. Provisional definitions

### Definition 1. Host frame
A host frame is a finite rigid oriented chamber system equipped with a lawful transport grammar on states.

### Definition 2. Admissibility frame
An admissibility frame is a host frame relative to which larger constructions are generated, embedded, or rendered legible.

### Definition 3. Occupant graph
An occupant graph is a richer graph-theoretic or lens-visible construction whose organization is governed by a host frame.

### Definition 4. Host-induced legibility
A construction is host-legible if its local motion, orientation, and closure properties can be expressed coherently in the grammar of the host frame.
